01. What kind of backend work do you do?

I specialise in Java & Spring Boot microservices — REST API design, event-driven architecture with Kafka, containerisation with Docker & Kubernetes, and PostgreSQL/Redis data layers. I'm comfortable owning a service end-to-end, from schema design to CI/CD pipeline.

02. What's your preferred tech stack?

Day-to-day: Java 17, Spring Boot 3, Hibernate, PostgreSQL, Redis, Kafka, Docker. For infra I reach for Kubernetes and AWS (EC2, RDS, S3). On the frontend I can pair with React when full-stack ownership is needed.

03. Are you open to freelance or contract work?

Yes — I take on select freelance projects alongside my main role, particularly API design, performance optimisation, and system architecture consulting. Reach out with your scope and timeline and we'll see if it's a fit.

04. How do you approach system design?

I start from requirements and failure modes first — what breaks, what scales, what can be async. I favour simple, well-bounded services over clever architectures. Documentation and observability (metrics, structured logs, tracing) are first-class concerns, not afterthoughts.

05. What does your development process look like?

Iterative and test-driven. I prototype the data model and API contract first, write integration tests against a real DB, then layer in the business logic. Every service ships with a health endpoint, structured logging, and a Docker Compose for local dev from day one.
